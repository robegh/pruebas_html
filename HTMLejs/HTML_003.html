Define un conjunto de enlaces de navegación.

Elemento de bloque Main:

Especifica el contenido principal de un documento.
El contenido dentro del elemento Main debe ser exclusivo del documento. No debe contener ningún contenido que se repita en documentos como barras laterales, enlaces de navegación, información de derechos de autor, logotipos del sitio y formularios de búsqueda.
No debe haber más de un elemento Main en un documento. El elemento Main no debe ser descendiente de un elemento Article, Aside, Footer, Header o Nav. Se aconseja usar CSS para darle estilo a Main.

Elemento de bloque Article (HTML5):

Especifica contenido independiente y autónomo.
Un artículo debería tener sentido por sí solo y debería ser posible distribuirlo independientemente del resto del sitio.
Fuentes potenciales para el elemento Article: Publicación en el foro, Entrada en el blog, Noticia.
El elemento Article no se representa como algo especial en un navegador. Sin embargo, puede usar CSS para diseñar el elemento Article.
Un elemento Article podría tener un Header con el título, un Footer (fecha de publicación, autor e información adicional) y un cuerpo con el detalle del contenido.
Si se trata de un artículo (Article) muy completo, se podría estructurar la información por secciones (Section), cada una con su Header y contenido, por ejemplo.
Se pueden tener varios Article dentro de otro elemento Article (subartículos), o bien englobarlos mediante un elemento Section (por ejemplo, para un listado de noticias).
Se aconseja usar CSS para darle estilo a Article.

Elemento de bloque Section (HTML5):

Define una sección en un documento.
Se usa para representar un bloque de nuestra página que tiene valor semántico.
Un elemento Section puede contener varios elementos Section (subsecciones).
Un elemento Section puede contener varios Article y viceversa.
Si tenemos que clasificar por la importancia del significado, el ranking sería:
1. Article: sería la etiqueta que contiene la información más relevante.
2. Section: la que contiene información menos relevante.
3. Div: contiene información que no aporta significado ninguno.
Se aconseja usar CSS para darle estilo a Section.

Elemento de bloque Header (HTML5):

Representa un contenedor de contenido introductorio o un conjunto de enlaces de navegación.
Un elemento Header normalmente contiene: uno o más elementos de encabezado (h1-h6), logo o icono de información de autoría.
Se puede tener varios elementos Header en un documento HTML (dentro de Body: sería el encabezado de la página en su conjunto; y en Article, Section, Aside o Nav). Sin embargo, Header no se puede colocar dentro de un Footer, Address u otro elemento Header.
Se aconseja usar CSS para darle estilo a Header.

Enlace al ejemplo: Ejemplo de elemento de bloque Nav, Main, Article, Section y Header juntos

Elemento de bloque Footer (HTML5):

Define un pie de página para un documento o sección.
Un elemento Footer normalmente contiene: información de autoría, informacion registrada, información del contacto, mapa del sitio web, enlaces de volver hacia arriba o documentos relacionados.
Se puede tener varios elementos Footer en un documento HTML (dentro de Body: sería el pie de la página en su conjunto; y en Article, Section, Aside o Nav, entre otros). Sin embargo, Header no se puede colocar dentro de un Footer, Address u otro elemento Header.
Footer no puede ser descendiente de Address, Header o de otro elemento Footer.
Consejo: la información de contacto dentro de un elemento Footer debería ir dentro de una etiqueta Address.
Se aconseja usar CSS para darle estilo a Footer.

Enlace al ejemplo: Ejemplo de elemento de bloque Nav, Main, Article, Section, Header y Footer juntos

Elemento de bloque Aside (HTML5):

Define algún contenido a un lado del contenido en el que se coloca.
El contenido Aside debe estar relacionado indirectamente con el contenido circundante pero con una importancia menor.
Aside no puede ser descendiente de Address.
Consejo: El contenido Aside suele colocarse como una barra lateral o como inserciones en un documento, y contienen una explicación al margen como una definición de glosario, elementos relacionados indirectamente, como publicidad, la biografía del autor, o en aplicaciones web, la información de perfil o una lista de enlaces a blogs relacionados, etc.
El elemento Aside no se representa como algo especial en un navegador. Sin embargo, se puede usar CSS para diseñar el elemento Aside.
Se aconseja usar CSS para darle estilo a Aside.
